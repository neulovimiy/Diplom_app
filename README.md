# Decision Support System for Information Resource Valuation

Веб-приложение для оценки ценности информационных ресурсов предприятия, анализа рисков и моделирования влияния инцидентов на их критичность.

Проект реализует интеллектуальную систему поддержки принятия решений (СППР) с использованием локальных LLM-моделей и векторного поиска документов.

---

## Общее назначение

Система предназначена для:

- оценки ценности информационных ресурсов
- расчёта интегрального ранга критичности (1–9)
- анализа рисков по нескольким критериям
- моделирования влияния инцидентов
- хранения истории оценок
- формирования аналитических отчётов

Дополнительно реализован ИИ-ассистент для автоматической оценки параметров ресурсов и экспертных рекомендаций.

---

## Основной функционал

### Управление ресурсами
- регистрация информационных ресурсов
- хранение характеристик в базе данных
- автоматическое определение параметров через ИИ

### Расчёт ценности
- бонусная модель оценки параметров
- 5 групп риска: финансовый, операционный, юридический, репутационный, стратегический
- нормализация значений
- итоговый ранг (1–9)

### ИИ-ассистент
- автоматический анализ описания ресурса
- генерация 9 параметров через LLM
- экспертные объяснения расчётов
- анализ инцидентов и их влияния
- рекомендации с обоснованием

### Динамика и инциденты
- графики изменения ранга во времени
- моделирование инцидентов
- пересчёт параметров после событий
- фиксация истории изменений

### Отчётность
- экспорт данных в CSV
- генерация PDF-отчётов
- статистика по ресурсам
- история оценок

## Математическая модель оценки

### 1. Базовый ранг
Определяется категорией доступа ресурса.

### 2. Бонусная система
Каждый параметр увеличивает или изменяет вклад критериев риска.

Пример:
- критичность → +финансы, +операции, +стратегия
- масштаб → +операции, +стратегия
- отсутствие бэкапа → штраф к операционному риску

### 3. Нормализация

S = (R-1)/9 для части критериев риска

S = (R-1)/7 для части критериев риска

### 4. Итоговый ранг
Рассчитывается по суммарному индексу риска SΣ (0–5) и отображается в шкале 1–9.

## Как работает ИИ-система

### 1. Автоматическое определение параметров
- анализ текста ресурса
- поиск релевантных документов через FAISS
- формирование контекста
- генерация JSON с параметрами

### 2. Экспертные объяснения

LLM генерирует:
- анализ каждого критерия риска
- текстовое обоснование (7–10 предложений)
- ссылки на нормативные документы

### 3. Анализ инцидентов
- обработка текстового описания события
- оценка влияния на каждый тип риска
- изменение рангов (+2 / -2)
- фиксация результата

---

## RAG-система (база знаний)

- индексация документов (PDF, DOCX, TXT, RTF)
- разбиение на фрагменты (chunking)
- хранение эмбеддингов в FAISS
- retrieval 8–20 релевантных фрагментов
- использование в prompt для LLM

снижает галлюцинации модели и повышает обоснованность ответов

## Основные вкладки приложения

1. **Регистрация ресурса** - ввод данных, ИИ-предложения параметров, сохранение в БД
2. **Анализ ресурса** - расчёт ранга, экспертные рекомендации, корректировка параметров
3. **Динамика** - графики изменений, моделирование инцидентов, пересчёт оценок
4. **База и отчёты** - таблицы ресурсов, статистика, экспорт CSV / PDF
